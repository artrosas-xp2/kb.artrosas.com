## A note on scope

These docs cover **dual-interface smart card + FIDO2 tokens** of the kind
typically deployed in enterprise IAM rollouts as a replacement for legacy
OTP fobs. Specific vendor names and proprietary command syntaxes have
been generalized; the underlying concepts (ACA, PIV, OATH, FIDO2 CTAP)
are open standards that apply across the category.

If you're rolling out keys at scale and finding the vendor docs read
like an API reference rather than a help desk playbook — this is the
gap I'm trying to fill.
